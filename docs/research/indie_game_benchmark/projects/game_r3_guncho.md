# GUNCHO 深度产品分析

研究日期：2026-07-22（Asia/Shanghai）

类型：小型六角格射击战术肉鸽

证据：开发者周年复盘、开发者控制的 Google Play 页面及开发者上传实机视频；未下载或运行游戏。

## 产品与核心循环

GUNCHO 把左轮弹巢做成战场规则：弹巢中的格位与角色周围的六个方向一一对应，移动、射击和退弹会改变下一发可射方向。官方页面还列出随机生成的短局、不同敌人、子弹升级、战术能力、Easy/Expert、日常挑战、解锁、成就和云存档。开发者复盘说明团队用约 11 个月完成首发；Puzzle Mode 会公开敌人行动顺序，进一步把试错局面变成可推演谜题。

[官方实机 00:20](https://www.youtube.com/watch?v=YaU11dBtIsY&t=20s) 可见：小型六角竞技场保持在一屏，底部弹巢 UI 大到足以和敌人方向并读；一次动作结束后，单位位置、弹巢朝向和可继续判断的局面很快稳定下来。其价值不是“西部皮肤”，而是让资源状态、空间方向与敌人意图共享一套可见语法。

## 表现：特效、Shader、动效

- 开发者明确称这是工作室首款完整 3D 游戏，美术与动画由 Terri Vellmann 负责；实机以斜俯视小场景、低复杂度轮廓和有限单位数量维持读盘优先级。
- 枪击用短位移、枪口闪光、尘屑和局部受击反馈收束结果，没有观察到长时间遮挡棋盘的全屏特效。
- 一周年文章没有公开 Shader、后处理管线、粒子预算或补间参数。由画面风格反推“低成本 Shader”并不可靠，故不作实现结论。
- 相机曾与弹巢理解发生冲突：早期让旋转视角连带改变方向参照，后来改成明确的左右旋转控制。这是一手的“视觉参照系必须服务规则”的删改证据。

## 音效

开发者复盘明确列出 Sam Webster 负责 soundtrack，并说明 2025 年内容更新为三名新角色各加入一首曲目。预告中枪声、退弹/装填和命中反馈短而可区分，持续配乐负责主题气氛。预告混音不能证明移动设备外放清晰度、同时发声上限、响度规范或独立音量控制；这些均未证实。

## UI/UX 与可访问性

- 弹巢不是角落里的装饰计数器，而是持续可见的主状态；玩家无需先开详情面板才能理解下一枪方向。
- Puzzle Mode 公开敌人行动次序，属于把确定性信息交还给玩家，而不是降低规则复杂度。
- PC 版支持鼠标和手柄，官方复盘称 Steam Deck Verified；移动版则用明确左右按钮处理旋转。跨输入已被产品化，但按键重绑、色觉替代、减少动效和屏幕阅读语义未从一手页面证实。
- 当前项目的方向输入与棋盘坐标已经分层；GUNCHO 的启示是给这种不变量补可见反馈与跨视口回归，而不是再造第二套坐标系统。

## 玩法变体与功能设计

随机关卡承担重复游玩，Easy/Expert 提供压力档位，Daily 提供同题比较，Puzzle Mode 则把战斗规则转成显式顺序谜题。解锁、成就和云存档承接中期目标。开发者还在首年更新中加入三名角色及对应音乐，说明角色差异可以复用同一基础语法扩展，而不必持续增加操作按钮。

当前 2048 项目已有 6 种模式、可变/稀疏棋盘、稳定种子、撤销、回放和主题表现，系统广度更强；缺口是公开意图、局内少量选择、Daily 资格以及“方向状态跨相机/布局仍一致”的产品级验证。

## 性能与低端线索

官方确认移动端、PC 和 Steam Deck 覆盖，且小竞技场、少量同时单位和短促反馈是受控的呈现范围。但没有公开帧时间、内存、draw call、低端机型或热量数据；“小盘 3D”不能直接等于性能优秀。若借鉴，应继续以当前 16.667 ms 帧预算、预热和池化指标实测，不把竞品平台覆盖当性能基准。

## 与当前项目比较及 GF 映射

| 发现 | 当前项目判断 | GF / 所有者 |
| --- | --- | --- |
| 弹巢方向与世界六向一一对应 | 可借“单一可见资源与空间绑定”，不复制六角枪械规则 | 规则、方向与资源状态归项目；回合编排用 `GFTurnFlowSystem`，结算动作用 `GFActionQueueSystem` |
| 相机旋转曾破坏规则参照系 | 当前已有模型坐标/视口分离，应补跨布局、旋转、输入的联合回归 | Viewport 与输入设施可复用；canonical 坐标及映射契约归项目，不在 GF 建第二真源 |
| Puzzle Mode 公开敌人次序 | 当前回放强，但行动前意图较弱 | 预览必须查询同一确定性规则；overlay 归项目，历史/回放沿用现有链路 |
| Daily、排行榜、角色解锁 | 当前稳定 seed 足够支撑轮换题，但资格和服务边界未完成 | seed、SaveGraph 可复用；题目发布、资格、排行权威归项目/平台 Adapter |
| 短枪击反馈保持盘面清晰 | 当前已有 Shader、震动和声音基础，可统一强度档位 | 参数走 `GFShaderParameterUtility`，预热走 `GFRenderWarmupUtility`，音频走 `GFAudioUtility`；具体表现归项目 |

## 可借鉴点、风险与证据限制

最值得做的是原创的“单敌人意图 + 行动后方向状态”垂直切片，以及相机/棋盘/HUD 参照系回归；其次才是小型战术变体。不要把六角格、左轮弹巢、角色能力或具体关卡直接移植到 2048，也不要因为存在 GF 空间相关包就静默新增依赖。视频是剪辑素材，未证明输入延迟、动画可取消性、教程完成率或真实性能；实现前仍需本项目原型与指标验证。

## 许可证与复用边界

GUNCHO 是商业闭源项目。开发日志、商店页和视频用于事实核验，不构成代码、Shader、美术、角色、音乐、声音或关卡的再利用许可。本报告只提炼“状态与空间一一对应”“公开意图”“稳定参照系”等抽象产品模式。
