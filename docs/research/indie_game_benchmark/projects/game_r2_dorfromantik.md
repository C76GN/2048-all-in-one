# Dorfromantik 深度产品分析

研究日期：2026-07-22（Asia/Shanghai）
类型：六角地块放置 / 轻策略高分
证据：开发者官网、官方 press kit 与开发者上传的实机预告；未下载、安装或运行游戏。

## 产品与核心循环

Dorfromantik 从有限牌堆逐张抽取六角地块，玩家旋转并把地块接到不断扩张的地图边缘。森林、村庄、田野、铁路和水道等边缘是否正确衔接，会影响分数、任务推进和后续可用牌；完成任务可补充牌堆，因此“眼前完美连接”和“为未来保留接口”持续冲突。官方把产品定位为放松的建景体验，同时保留战略性高分目标。

开发者列出的公开模式不只经典局：Quick、Hard、Custom、Creative、可分享的自定义规则，以及使用统一设置与 seed 的月度模式。可解锁地块与生物群系又把重复游玩转成长期收藏目标。这个结构值得当前项目关注，因为它没有靠新增一套输入语法延长寿命，而是围绕“放一块”改变资源压力、目标与内容分布。

官方 Full Release Trailer 在 [00:55](https://www.youtube.com/watch?v=gc8kT3WciJs&t=55s) 展示实际盘面：已生成的乡野地图外缘是一圈白色待放置边界，下一块牌独立显示在右侧，地图上保留小型任务标记，右上角显示总分。玩家在一个视野内即可回答“能放哪里、接下来放什么、局部目标在哪、总体表现怎样”。

## 表现：特效、Shader 与动效

- 大地图依靠村庄、林地、田野和水面的轮廓与大色块分区，远景仍能识别地貌；白色未填边界把交互层从手绘景观中提出来。
- 下一牌与地图保持空间分离，任务标记尺寸较小，视觉权重从“当前落点”逐级下降到局部目标和总分。
- 官方称其具有手绘桌游式观感。可以借鉴的是统一的材质语言和安静背景，而不是复制具体地块、配色、模型或纹理。
- 视频能确认落牌后景观连续、地图存在细微环境活动，但不能判断地块生长、阴影、景深或水面由 Shader、粒子、骨骼动画还是预制动画实现，也不能量化 easing 与反馈时长。

对当前项目最直接的表现启发，是把“可操作范围”变成稳定的棋盘内层，而把成功落位的仪式限制在局部：边缘短亮、相邻关系逐段确认、目标标记轻响应。现有反馈能力已足够，不需要再增加全屏特效或第二套 Tween/队列系统。

## 音效

官方 press kit 对音乐与声音有明确署名：音乐涉及 Laryssa Okada、OnlySound 与 pygoscelis，声音署名 pygoscelis。这只能证明创作归属，不能证明自适应分层、混音优先级或具体触发规则；本研究没有独立试听，也不据预告片评价落牌、任务完成和环境声的响度关系。

若当前项目引入空间摆放，应按语义拆分“选中、旋转、合法预览、非法预览、落位、完美匹配、任务完成”，继续交给现有音频事件/Bank 和主题资源承载。舒缓并不等于无反馈：合法确认需要短而明确，环境或 BGM 应让位于决策声，不另建音频管理器。

## UI/UX

- 下一牌、开放边界、任务和总分同时可见，减少为确认规则而离开棋盘。
- 未填边界不是单纯装饰，它把不断变化的合法交互区域外显；若有旋转或边缘匹配，预览还应说明哪几条边满足、哪几条边冲突。
- 远景地图的可读性来自类别轮廓和色块，不应只靠相近色差。当前项目深化可访问性时，合法性还需用线型、符号或边缘形状编码。
- Quick、Hard、Custom、Creative 与月度模式让不同意图各有入口；比在同一设置页堆叠大量开关更容易形成清晰预期。

当前项目已有真实响应式布局、抽象输入和焦点机制。若做放置试验，鼠标悬停、触控拖放、键盘/手柄逐格导航必须映射到同一组抽象动作；相机缩放、旋转与确认不能形成仅鼠标可用的旁路。

## 玩法变体与功能设计

有限牌堆把每次落位变成不可无限修正的承诺；边缘匹配为局部评分，任务为中期方向，补牌奖励延长整局，解锁内容与月度 seed 则提供局外回访理由。四层目标共享一块盘面，所以复杂度来自关系而非按钮数量。

对 2048 当前项目，最适合迁移的是两个小切片：

1. 在棋盘编辑器或独立实验模式中，加入“下一构件 + 合法边界 + 相邻关系预览”，落位仍进入现有确定性历史。
2. 建立固定规则版本、公开 seed 和期限标签的月度挑战；结果可离线保存并附回放，首版不声称线上权威排行。

不建议把六角地貌、铁路/河流任务或同类美术题材直接搬入，也不建议为了这一灵感把 GF 变成通用六角地图框架。

## 性能线索

官方公开页面能证明 PC、Nintendo Switch 等发行形态和可能持续扩大的地图，但没有帧率、帧时间、内存、加载时间、最大地块数或低端设备数据。由大地图画面推测“开放边界集合、邻接索引、视口裁剪和远景简化值得关注”属于风险推断（D），不是对原作内部实现或性能的结论。

当前项目若做原型，应只在一次确定性落位后增量更新六个相邻关系、开放边界与受影响任务；相机外对象采用既有可见性/池化路径，记录最坏地块数下的 16.667ms 帧预算。预览计算不得推进 RNG 或污染回放状态。

## 与当前项目比较及 GF 映射

| 发现 | 当前项目判断 | GF / 正确所有者 |
| --- | --- | --- |
| 下一牌、合法边界、局部任务、总分同屏 | 当前移动反馈清楚，但缺承诺前的空间方案预览 | 预览规则归 `gameplay` / `board_editor`；UI 复用现有 route、focus 与响应式布局 |
| 有限牌堆形成明确终点 | 当前主要围绕目标值/分数，缺正式有限关卡 | 牌堆与胜负属于项目规则；确定推进复用 `GFTurnFlowSystem` |
| 月度统一 seed 与规则 | 当前已有自定义 seed、稳定 hash 和回放，缺挑战资格模型 | `GFSeedUtility`、SaveGraph、回放提供机制；日期边界、规则版本、奖励与排行归项目/平台 Adapter |
| 落牌后的局部仪式 | 当前 Shader、粒子、主题和音效机制较强 | 复用 `GFActionQueueSystem`、`GFShaderParameterUtility`、`GFAudioUtility`；视觉配方由主题拥有 |
| 地块/生物群系解锁 | 当前内容包、图鉴和成就可承载，缺摆放内容目录 | 内容数据归项目资源；持久进度写入 SaveGraph，不另写存档 |
| 持续扩张地图 | 当前棋盘规模有界且已有池化/预热基础 | 先做项目级邻接索引、可见性和压力测试；没有跨项目证据，不认定 GF 缺口 |

## 可执行建议、风险与证据边界

- **P0**：在棋盘编辑器加入只读“合法落点与受影响邻接”预览，覆盖撤销、重做、书签、回放和三类输入；预览不得改变权威状态。
- **P1**：用现有 seed 与回放做一个离线月度挑战切片，保存挑战 ID、规则版本、seed、完成时间和资格标记；先验证跨平台确定性。
- **P2**：验证大盘面的远景简化、任务标记聚合与视口外更新预算，仅在数据证明需要时下沉通用机制。

产品页和官方视频没有向本项目授予代码、Shader、音频、美术或关卡复用许可；按商业内容不可复用处理。本报告只借鉴公开的交互原则，没有运行游戏或反推内部算法。视频画面可以支持 HUD 与空间关系观察，不能支持性能、技术栈或具体动效实现结论。
