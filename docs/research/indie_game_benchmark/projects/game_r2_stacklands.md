# Stacklands 深度产品分析

研究日期：2026-07-22（Asia/Shanghai）
类型：卡牌堆叠合成 / 生存与轻量战斗
证据：开发者 itch.io 产品页、开发日志与开发者上传的实机视频；未下载、安装或运行游戏。

## 产品与核心循环

Stacklands 把卡牌同时用作村民、资源、建筑、食物、敌人、配方提示和计时生产设施。玩家把村民叠到浆果丛上采集食物，把资源和村民叠在一起建造，再出售卡牌获得金币、购买补充包；每个 Moon 结束时必须喂养村民。空间整理、配方发现、时间推进和生存经济因此都发生在同一张桌面上。

作者产品页列出 200 多张卡牌、60 多项 Ideas、50 多个任务和 13 种卡包。Island 更新又加入独立岛屿棋盘、100 多张新卡、状态效果和更多系统。规模增长的关键并不是单纯加卡，而是官方持续补足“如何移动一整叠、如何在拥挤桌面定位、何时自动暂停、如何解释配方”的操作层。

官方 Steam 视频在 [00:15](https://www.youtube.com/watch?v=7qpT2xXUaCs&t=15s) 展示实际盘面：浅绿色桌面上有重叠卡牌，右侧常驻蓝色 `House` Idea 卡；字幕直接在盘面上解释两块木头、一块石头和一名村民如何开始建造房屋。配方说明与对象没有被拆到另一套百科界面，学习发生在操作现场。

## 表现：特效、Shader 与动效

- 统一尺寸的卡牌以颜色、标题、插图和层叠位移表示类别与堆关系；桌面背景克制，让大量对象仍保持可选中轮廓。
- 拖动目标高亮、卡牌吸附到网格、生产队列和卡包揭示都服务于“对象会去哪里、现在正在做什么”。官方 Island 更新明确提到拖动目标高亮与可重绑定的网格吸附；实现技术没有公开。
- 开发日志明确提到在相机锁定/过场时加入 vignette，这能说明产品用边缘暗化标识控制权变化，但不能证明具体 Shader 或性能成本。
- 视频和文字不足以量化拖放阈值、吸附 easing、堆叠展开速度、生产进度动画或粒子数量；也不能断言这些反馈由 Shader、Tween 或节点动画实现。

对当前项目，值得借鉴的是“直接在潜在目标上给反馈”，例如拖动构件时显示可接受对象、需求与预计产物。已有 `GFActionQueueSystem` 可以承载确认后的短序列；拖动预览应保持即时，不应排进阻塞业务输入的长动画队列。

## 音效

产品与日志没有提供足以验证的音频结构或混音数据，已观察视频也没有被用来独立评价拾取、叠放、卡包、生产或战斗 SFX。不能从卡牌动作频繁就外推其内部音频限流方案。

如果当前项目实验空间配方，应为“拾取、越过合法目标、非法释放、吸附、生产开始、生产完成”定义语义事件，并在同帧大量完成时使用现有音频能力做合并/节流。主题提供素材，`GFAudioUtility` 提供机制；不复制原作声音，也不创建平行播放器。

## UI/UX 与可访问性

Stacklands 的公开开发日志提供了罕见且可执行的迭代证据：

- v1.0.9 支持按 Shift 移动整叠卡、`+/-` 缩放、空格结束 Moon、最大缩放覆盖整张桌面，并修复战斗卡住等问题。
- v1.0.10 增加数字键速度切换，并允许在暂停时买卖，减少实时压力与重复点击。
- Island 更新允许生产设施排队多张卡，提供可重绑定的吸附到网格、拖动目标高亮、多存档槽、扩写后的 Idea 说明、控制器支持和按键重绑定。
- 同一更新加入“拖动时自动暂停”的可访问性选项，还调整标签、转场、进度保存和多处 UI。

这些条目说明复杂空间对象需要成套的可逆与替代输入，而不是只增加一个拖动手势。当前项目若做任何卡/块摆放，最低验收应同时覆盖：移动单个与整组、取消拖动、暂停、键盘/手柄焦点、触控命中、缩放后相机跟随、重绑定和存档恢复。

## 玩法变体与功能设计

Idea 卡把发现过的配方保留为局内/局外知识，卡包将经济和内容分发绑定，Moon 计时给空间整理施加压力；独立岛屿棋盘则让扩展内容拥有新的资源与状态，却仍沿用“叠卡触发”这一主语法。生产排队是在不改变核心动作的前提下降低重复操作，是很好的后期易用性增量。

当前 2048 项目不适合复制完整村民、生存、经济或战斗。更安全的迁移是一个有界的“空间配方实验”：两到三种项目自有构件在相邻或重叠时给出产物预览，确认后作为单条确定性命令进入历史；Idea 式说明记录玩家已经发现的项目规则。这个试验可放在编辑器/实验模式，不污染六个既有模式。

## 性能线索

官方更新能证明对象数量、重叠、独立棋盘、生产队列和缩放是实际产品问题域，但没有公开帧率、帧时间、内存、对象上限或性能剖析。由此推断“全表扫描命中、每帧重算所有堆关系、每个计时器独立更新可能形成峰值”属于机制风险（D），不是原作性能结论。

当前项目若原型化，应给每个空间对象稳定 ID，用有限空间索引获取候选目标，只在拖动对象与邻近堆变化时重算；生产/反馈事件批量推进，保存逻辑记录语义关系而非临时坐标。需分别测最大重叠、连续拖动、同帧多完成、缩放全景和恢复存档，守住 16.667ms 帧预算与 50ms 输入反馈目标。

## 与当前项目比较及 GF 映射

| 发现 | 当前项目判断 | GF / 正确所有者 |
| --- | --- | --- |
| 卡牌既是对象又是配方/生产输入 | 当前有 tile catalog 与自定义棋盘，缺局内 build 和空间配方 | 卡/构件、配方、目标索引归项目 gameplay；不新增 GF 通用卡牌经济系统 |
| 配方解释与可操作对象同屏 | 当前说明页较多，缺落位前的需求/产物提示 | 项目 tooltip/overlay 承载；复用现有 UI route、focus、text fitting |
| 拖动时自动暂停、暂停仍可买卖 | 当前确定性动作和历史强，但没有实时拖放压力 | 暂停策略归项目；命令推进复用 `GFTurnFlowSystem`，不把 UI 拖动写进领域 RNG |
| 整叠移动、网格吸附、目标高亮 | 当前稀疏棋盘与编辑器是合适实验面 | 抽象动作复用 `GFInputMappingUtility`；`GFDragDropUtility` 只可复用会话、落点和命中生命周期，拖放/吸附/配方语义仍归 `board_editor` / gameplay |
| 多存档槽、进度与配方知识 | 当前已有 SaveGraph、图鉴、成就 | 新 section/资源版本复用 SaveGraph；禁止另写 JSON 存档旁路 |
| 大量重叠对象与生产队列 | 当前池化/预热有基础，未验证任意重叠压力 | 项目先做稳定 ID、空间索引与基准；只有可复用证据充分时才反馈 GF |
| 局部吸附与生产完成反馈 | 当前动作队列、Shader/音频能力齐全 | `GFActionQueueSystem`、`GFShaderParameterUtility`、`GFAudioUtility`；配方视觉归主题 |

## 可执行建议、风险与证据边界

- **P0**：在棋盘编辑器做一个不进入正式经济的空间配方只读预览，明确显示需求、产物、合法目标和失败原因，并覆盖撤销/重做/回放。
- **P1**：补一套拖放可访问性验收：拖动自动暂停开关、取消、整组移动、焦点等价操作、触控命中和按键重绑定。
- **P2**：用合成压力场景验证稳定 ID、局部索引和同帧事件合并；数据不足时不把对象系统下沉 GF。

Stacklands 是商业产品，所用页面没有向本项目授予代码、卡面、美术、音效、配方或数值复用许可；全部按不可复制处理。本报告没有运行游戏，版本日志能证明公开功能与修复，不能证明内部架构；视频仅支持所述时间点的画面事实。
