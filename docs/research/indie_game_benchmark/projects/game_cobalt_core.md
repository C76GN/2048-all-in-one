# Cobalt Core 深度产品分析

研究日期：2026-07-22（Asia/Shanghai）
类型：单轴太空战 / 牌组构筑肉鸽
证据：开发商官网、开发团队官网与发行商上传的官方实机视频；未下载或运行游戏。

## 产品与核心循环

Cobalt Core 把飞船战斗压缩在一条横轴：玩家用卡牌左右移动船体，躲避导弹、让敌方攻击落空或把武器舱对准弱点。官方页面明确列出数百张卡牌及分叉升级、三名船员混合牌组与剧情、数百种角色/飞船组合、可调难度、每日挑战和时间循环叙事。

官方 Launch Trailer 在 [01:32](https://www.youtube.com/watch?v=OE8_dfMhlfo&t=92s) 展示实际战斗 HUD：敌我飞船沿同一水平轴对齐，顶部显示敌方意图/攻击位置，底部是手牌和分段生命/护盾，方向箭头把位移选择直接映射到战线。另一个已观察点 [00:41](https://www.youtube.com/watch?v=OE8_dfMhlfo&t=41s) 是蓝色水平拖影遮罩转场，可验证其横向运动母题，但不应当作玩法证据。

## 表现：特效、Shader、动效

- 船体、攻击轨迹、意图和卡牌分居清晰层级；背景太空氛围不与战斗轴竞争。
- 00:41 的水平拖影/擦除把“横移”这一核心动词延伸到场景转场，形成一致的 motion language。实现方式无法由视频确定，不应擅自认定为某种 Shader。
- 01:32 的状态信息依赖箭头、分段条、图标和空间位置多通道编码；这比单纯用颜色提示攻击更适合可访问性深化。

## 音效

开发团队官网明确署名 Aaron Cherof 负责音乐；本轮没有独立试听，不能评价配乐如何随战斗状态变化，也不能确认 SFX 混音。若借鉴其时间循环/危险状态，本项目现有 `GFAudioBank` 已支持 BGM、ambient、state/switch 与 crossfade，应先扩充主题内容和语义状态，不新建播放器管理器。

## UI/UX

- 敌方意图与攻击位置在出牌前显示，位移卡因此具有可预测的空间价值。
- 手牌、能量/状态、船体/护盾和敌方信息围绕单一战线排列，避免传统二维战术地图的摄像机和寻路负担。
- 卡牌分叉升级为同一内容提供方向性选择，比只提高数值更容易形成 build 身份；但 tooltip 必须说明升级差异、目标格和结算顺序。
- 每日挑战与可调难度给不同熟练度玩家明确入口，正好对应当前项目已确认的产品缺口。

## 玩法变体与功能设计

三名船员各带一组牌和剧情，组合后共享一艘船；飞船本身再改变几何与基础能力。每日挑战用共同条件提供可比较短目标，时间循环解释重复尝试。对本项目可迁移的是“公开 seed + 固定挑战条款 + 可验证回放”，以及让少量模块改变同一滑动规则，而不是复制卡牌或太空题材。

## 性能线索

单轴战线主动限制单位位置和相机状态，可能降低预测与绘制复杂度（D）；官方发行页面覆盖桌面/主机形态，但未给帧率、帧时间、内存或加载数据。卡牌手牌、角色表情、背景、导弹和转场仍可能产生 UI/动画峰值。若做每日挑战，重点性能风险反而是启动、seed 校验、回放序列化和历史列表，不应只测战斗帧率。

## 与当前项目比较及 GF 映射

| 发现 | 当前项目判断 | GF / 所有者 |
| --- | --- | --- |
| 单轴位移直接规避/对准攻击 | 当前有方向滑动，但缺敌方意图和目标弱点 | turn flow/seed 复用 GF；攻击、弱点与位移牌属于 gameplay |
| 每日挑战 + 可调难度 | 当前明确缺每日挑战，已有确定性 seed/回放 | 日期解析为公开 seed、结果进 SaveGraph；平台排行经 `platform_runtime` Adapter |
| 三船员混合牌组 | 当前内容包/模式可组合，缺局内 build | 先做项目级挑战 modifier；不新增通用 GF 卡牌系统 |
| 分叉升级 | 当前成长多为图鉴/成就，缺局内选择 | 选择状态和重放命令归项目；持久化复用 SaveGraph |
| 横移母题贯穿转场与战斗 | 当前 CMYK 主题有一致视觉语言但参数分散 | 用项目 motion/feedback Profile + `GFActionQueueSystem`；Shader 经校验 Utility |
| 音乐明确署名、可能承载叙事气氛 | 当前音频机制强、素材与自适应层浅 | 优先扩 `GFAudioBank` state/switch/crossfade 内容，不建平行音频管理器 |

## 可借鉴点、风险与证据限制

P0 可直接形成“每日 seed + 固定模式/棋盘 + 本地可验证回放”的产品切片；首版不接线上权威排行，先验证跨平台确定性、日期边界、离线状态和作弊提示。P1 再测试一项分叉式局内奖励。不要复制卡牌、角色、飞船、剧情、图像或音频。本报告未运行游戏或试听音乐，视频观察只支持画面与 HUD；性能和实现技术均不能从宣传视频反推。
