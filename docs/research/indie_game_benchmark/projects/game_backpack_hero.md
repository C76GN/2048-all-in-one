# Backpack Hero 深度产品分析

研究日期：2026-07-22（Asia/Shanghai）
类型：空间背包构筑 / 回合肉鸽
证据：开发者 itch.io、官方开发日志与开发者上传的官方实机视频；未下载或运行游戏。

## 产品与核心循环

Backpack Hero 把传统肉鸽的装备栏变成空间谜题：物品不只占格，还会因邻接、方向、位置或同类组合改变效果；玩家可旋转和重排物品，再与敌人进行回合战斗。官方页面还列出不同英雄、数百种物品、随机地牢、商人/工匠，以及回到城镇后重建家园的长期结构。

1.0 官方视频在 [00:29](https://www.youtube.com/watch?v=5Xdo0OJYVw4&t=29s) 展示实际战斗：左侧聚光区域是一只塞入剑、食物、木块等物品的格状背包；底部玩家生命为 `55/55`，右侧两名敌人显示 `3/19`、`30/30` 和伤害意图，底部有 `End Turn`。物品布局、敌方意图与回合确认同时可见，直接把“整理背包”变成战斗决策面板。

## 表现：特效、Shader、动效

- 暗背景与聚光把注意力集中到背包；物品靠轮廓、朝向和格位区分，战场人物则退到辅助层。
- 敌人头顶的伤害意图和状态标记不需要播放完整动画就能传达下一回合后果。本项目可借鉴这种行动前反馈，而不复制背包或物品外观。
- 视频能确认像素画面、局部照明与界面切换，但不能识别照明是 Shader、遮罩还是预制素材，也不能量化物品拖动吸附、旋转和攻击动画时长。

## 音效

官方 itch 页面提供音乐/音频署名，但本轮没有独立试听，不能评价拾取、摆放、旋转、攻击或城镇环境的声音设计。对本项目更有价值的是把“选择、合法落位、非法落位、组合生效、结算”拆成不同语义事件，并让主题包提供素材；机制继续复用 `GFAudioBank`。

## UI/UX

- 背包格位同时是库存、build 和战斗控制面板，减少在装备页与战斗页之间来回切换。
- 敌方意图、生命和 `End Turn` 常驻，玩家可在承诺回合前调整布局。高组合复杂度要求 tooltip 清楚解释邻接、方向、消耗和叠加顺序。
- 官方页/开发日志列出教程、字幕和多平台/多语言支持；这为本项目当前缺少完整 onboarding 提供可比方向，但具体可访问性覆盖不能从功能列表完全确认。
- 拖放和旋转在鼠标、手柄与触控上的手感不同；若迁移空间组合思想，应先定义抽象动作与焦点路径，不写平台专用旁路。

## 玩法变体与功能设计

空间规则让同一件物品因位置而改变价值，天然产生“拿不拿、放哪里、是否旋转、为未来留多少空间”的多层取舍。英雄改变背包和物品语法，城镇重建提供局外目标。对 2048 项目可借鉴的是“配方占位关系”：例如自定义棋盘或主题构件按邻接形成加成；不必增加完整库存、装备稀有度和城镇系统。

## 性能线索

官方提供浏览器/桌面等发行形态可作为跨环境证据，但没有帧率、内存、加载或低端设备测量。大量物品邻接、旋转和触发顺序可能造成组合检查与 tooltip 更新成本，这是基于机制的风险推断（D），不是对原作性能的结论。本项目如实现相邻配方，应在一次状态变更后增量重算，并用诊断记录最坏布局和连锁峰值。

## 与当前项目比较及 GF 映射

| 发现 | 当前项目判断 | GF / 所有者 |
| --- | --- | --- |
| 位置、邻接、朝向共同决定效果 | 当前自定义棋盘/组合规则有基础，但缺空间配方 | 规则与索引属于 gameplay/board_editor；不要为单作灵感新增 GF 模型 |
| 敌方意图与背包 build 同屏 | 当前缺局内威胁和 build，但 HUD 可扩展 | 意图/物品数据归项目；UI 复用既有 route、focus、text fitter |
| 拖放、旋转、多输入 | 当前已走抽象输入，仍需焦点与触控验收 | 复用 `GFInputMappingUtility`/虚拟输入；不直调 `InputMap` |
| 教程、字幕和 tooltip | 当前 onboarding 与可访问性不足 | 教程状态归 navigation/gameplay；文本/焦点机制复用现有 GF 能力 |
| 城镇长期进度 | 当前已有成就、图鉴、统计，缺叙事型局外目标 | 若采用轻量目标，用 SaveGraph section；不另写存档 |

## 可借鉴点、风险与证据限制

P1 建议先在棋盘编辑器做“相邻构件配方”只读预览：不进入正式经济，不复制背包，验证触控/手柄焦点、undo/redo、保存和最坏重算成本。若价值不足即可丢弃。不要复制物品、英雄、城镇、图像、音频或数值；来源页面没有授予本项目这些内容的复用许可。本报告没有运行物品交互，拖放手感、音频和性能均未实测。
