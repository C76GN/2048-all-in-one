# Shotgun King: The Final Checkmate 深度产品分析

研究日期：2026-07-22（Asia/Shanghai）
类型：国际象棋变体 / 枪械回合肉鸽
证据：开发商 itch.io 与开发商上传的官方实机视频；未下载或运行游戏。

## 产品与核心循环

Shotgun King 保留棋盘、棋子走法和将军约束，却把玩家阵营压缩为一枚持霰弹枪的黑王。每回合在移动、射击、装填与收集灵魂之间取舍；过关后的卡牌会同时改变双方规则。官方页将其定义为“chess, but you replace your entire army with a royal shotgun”，并列出多难度、灵魂模式、无尽模式、成就、界面重做、可访问性、翻译和模组选项。

官方 v1.5 视频在 [00:42](https://www.youtube.com/watch?v=Kpnp4YPHYI0&t=42s) 显示实际棋局：黑王朝棋盘上瞄准，左栏把 `POWER 4`、`RANGE 3–5`、`ARC 55°` 常驻显示，右栏保留 `SOULS` 槽；瞄准方向和射界直接叠加在棋盘上。玩家因此不用离开主界面就能把武器参数转换成空间决策。

## 表现：特效、Shader、动效

- 视频呈现明显的 CRT/扫描线、轻微色偏与暗角语汇，像素棋子、枪口方向和高对比格线仍保持可辨认。
- 瞄准扇区把抽象数值变成空间预览，是比纯伤害数字更有用的动效层；本项目可对应为滑动路径、受影响格或连锁范围预览。
- 官方 itch 页专门为部分集成显卡的图形瑕疵提供 shaderless 启动批处理文件。这是一条重要反证：风格 Shader 不能成为可读性和可运行性的单点依赖。

## 音效

霰弹枪是产品核心，但本轮未单独捕获音轨，官方文字也不能证明射击、装填、棋子受击和界面的实际层级。因此不评价声音质感。若参考其动作节奏，本项目应让合并、连锁、目标达成和失败分别走语义事件与并发上限，不复制具体枪声音频。

## UI/UX

- 武器威力、射程和散布角始终与棋盘并列，降低“记住 build 后再估算”的负担。
- 瞄准预览在承诺行动前显示可能覆盖区域，符合高代价回合制游戏的可解释性要求。
- 官方页明确列出 reworked interface、accessibility 与多语言，但未细化减少动态、色盲或读屏覆盖；这些不能仅凭标签视为已验证完成。
- CRT 层提升辨识度的同时可能影响低视力、压缩视频和低端 GPU；必须能降级或关闭。

## 玩法变体与功能设计

产品用一条大胆的规则替换形成新核心，再通过双方卡牌、灵魂能力、多难度、无尽与模组扩大重玩。值得借鉴的不是“给 2048 加枪”，而是“只替换一个基础动词，并让所有 UI、反馈和成长围绕它重排”。例如某挑战把滑动改为有方向/距离成本的推送，仍复用既有格子、seed、回放和内容资源。

## 性能线索

官方对集成显卡瑕疵给出无 Shader fallback，说明实际发布中遇到过图形兼容性问题；这是 B 级一手性能/兼容线索，但没有硬件型号、帧率或修复范围。棋盘规模小、像素素材轻量只能作为推断（D）。本项目应把全屏纸张/印刷 Shader、局部反馈和无特效档分别纳入兼容测试，而不是只测开发机。

## 与当前项目比较及 GF 映射

| 发现 | 当前项目判断 | GF / 所有者 |
| --- | --- | --- |
| 射程/散布/命中区在行动前可视化 | 当前缺滑动后果与连锁范围预览 | 预览数据属于 gameplay；绘制走主题/棋盘反馈层，不改 GF |
| CRT 风格有 shaderless fallback | 当前有 Shader Profile 与预热，但降级矩阵尚不完整 | 继续用 `GFShaderParameterUtility`、`GFRenderWarmupUtility`；项目 settings/themes 提供关闭与低档 |
| 一枚主角重写熟悉棋盘 | 当前六模式较广，可尝试单动词垂直变体 | 回合调度/seed 复用 GF；规则、平衡、教程属于项目 Feature |
| 双方卡牌带来取舍 | 当前缺局内 build 与带代价选择 | SaveGraph 保存规范状态；奖励池和卡牌语义归项目 |
| 可访问性、翻译、模组 | 当前有中英与抽象输入，完整可访问性不足 | 输入复用 `GFInputMappingUtility`；不因竞品模组功能引入不可信脚本执行 |

## 可借鉴点、风险与证据限制

P0 借鉴是为高代价动作提供可关闭的空间预览，以及为所有风格 Shader 建无 Shader 退路。P1 才是用一条规则替换做独立挑战。CRT/扫描线与本项目 CMYK 纸媒主题可能产生噪声冲突，不应直接叠加。不要复制棋子造型、卡牌、文本、音频、图形滤镜或模组代码；官方页面没有提供可复用素材/代码许可证。本报告没有音频实测，兼容性结论仅限官方已披露的问题与 fallback。
