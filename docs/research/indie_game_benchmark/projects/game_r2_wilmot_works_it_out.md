# Wilmot Works It Out 深度产品分析

研究日期：2026-07-22（Asia/Shanghai）
类型：无盒图拼块 / 舒缓整理解谜
证据：发行商产品页、发行商新闻、官方补丁日志与发行商上传的实机预告；未下载、安装或运行游戏。

## 产品与核心循环

Wilmot Works It Out 让玩家逐次收到包裹中的方形拼块，再从颜色、边缘与局部图案推断它们属于哪幅画。与常见拼图不同，设计者在发行新闻中明确称自己喜欢“no-boxing”——没有盒面答案可照着完成。玩家先建立局部关系，再逐步发现整幅图像；新到货还可能包含暂时用不上的额外块，整理和记忆因此成为谜题的一部分。

Finji 产品页描述了七个房间、数十幅原创拼图、20 多种家居自定义，以及完成主流程后的 Marathon Mode。官方页面对拼图数量出现“60+”与“70”两种公开表述，本报告不强行统一成一个精确数字，只采用“数十幅”。宠物、房间布置和长期拼图进度为短时拼块操作提供温和的局外节奏。

发行新闻还给出开发过程的高价值证据：设计者从 Wilmot 的整理机制出发，一个月内原型已“感觉良好”，之后移除了升级与 Borky 等不服务核心的系统。它说明精品小型解谜的深化可以来自删除干扰、保持一个强动词，而不必为留存叠加经济层。

官方 AVAILABLE NOW 视频在 [00:25](https://www.youtube.com/watch?v=syfKNCYnz_s&t=25s) 展示实际盘面：米色空场中只有一组圆角方形拼块，局部蓝白图案已经组成一段人物/场景；Wilmot 自身也是同尺寸方块并位于拼图内部。画面没有盒面答案、分数或复杂工具栏，判断依据几乎全部来自拼块本身。

## 表现：特效、Shader 与动效

- 大面积低对比背景为彩色拼块留出安静工作区，圆角与统一尺寸让拼块身份稳定，内部图案承担主要信息。
- Wilmot 与拼块同样采用方形轮廓，使角色移动和“推/整理方块”的核心动词在视觉上统一；同时必须依靠面部/颜色等额外信号避免把角色误认成普通块。
- 完整答案被隐藏，但局部图案在相邻边上连续，形成无需文字的正反馈。可借鉴的是“局部关系充分、全局答案延迟”，不是复制画作、角色或具体配色。
- 已观察视频不足以确认拾取、推动、旋转、吸附、完成判定和庆祝动画的时长，也不能识别是否使用自定义 Shader、后处理或纯节点动画。

当前项目若借鉴这种克制表现，应让候选关系在对象边缘轻微对齐/高亮，确认后再做短吸附与完成仪式。提示层不应直接播放正确移动序列，避免把发现过程变成照抄操作。

## 音效

官方产品页将配乐描述为 jazzy electronic soundtrack，并署名 Eli Rainsberry。v1.0.43 补丁明确修复某些音乐轨不同步的问题，证明多音乐轨的同步是发布后真实维护点；它没有公开轨道数量、状态切换逻辑、响度、缓冲或混音实现。

本研究没有独立试听，因而不评价移动、吸附、包裹、完成或房间环境音。对当前项目的可执行含义是：如果主题音乐采用分层/多轨，必须把同步恢复、暂停、场景切换和设备失焦列为测试项；优先使用现有 `GFAudioUtility` / Bank 的 state、switch、crossfade 等机制，不建立第二套音乐时钟。

## UI/UX

- “无盒图”不是信息缺失，而是把信息从全局参考图转移到边缘、颜色、局部构图和已经完成的邻接关系。
- 空场与极少 HUD 降低视觉搜索负担；拼块越多时，则需要相机缩放、跟随和稳定对象身份支持。v1.0.43 明确修复了放大状态下的 camera follow。
- 同一补丁还修复 accessories 菜单需要多次按键、罕见拼块重复/消失等问题。这些都说明舒缓产品同样依赖可靠焦点、一次输入一次响应，以及持久化对象身份。
- 家居自定义和宠物是低风险奖励层：它们不改变拼图规则，却把完成成果映射到可见空间。当前项目可把类似原则用于主题/陈列解锁，不必增加资源经济。

对于当前项目，有限关卡的提示可分三层：先标出与当前目标有关的区域，再提示局部约束，最后才显示一项可行操作。每层都应被记录为明确的辅助使用状态，以便统计、挑战资格与回放解释一致。

## 玩法变体与功能设计

核心结构由“到货—整理—发现关系—完成图像—布置房间”组成。额外拼块让玩家暂时无法闭合当前图案，避免纯线性找边；七个房间提供章节节奏，Marathon Mode 再把内容组织为长时连续目标。设计者主动删除升级系统的公开反思尤其重要：当局内动词本身有发现感时，外加成长可能削弱而非增强体验。

当前 2048 项目最适合迁移的是“不给完整答案的有限关卡提示”，而不是完整拼图：

1. 为预设棋局提供局部关系提示，例如标出必须参与某次合并的区域、某个目标 tile 的约束或剩余步数压力。
2. 用短章节组织不同规则/拓扑，每章完成后解锁现有主题陈列或图鉴内容。
3. 加入连续谜题播放列表时，仍使用确定性命令和关卡 seed；提示、重开与撤销都进入资格记录。

## 性能线索

产品页与补丁能证明相机缩放/跟随、菜单输入和拼块身份是实际功能点，但没有帧率、帧时间、内存、加载或最大拼块数数据。由大量可移动拼块推断“命中检测、相邻匹配和缩放后文本/边缘清晰度需要压力测试”属于风险推断（D），不能当作原作性能问题。

当前项目实现有限关卡时，应保持规则状态与表现节点分离：拼块/tile 使用稳定 ID，提示只读权威状态，完成检测在一次命令后增量计算；相机动画与完成仪式不阻塞下一次状态保存。测试重点包括缩放后焦点位置、重复恢复、撤销后提示一致性，以及 50ms 内给出输入确认。

## 与当前项目比较及 GF 映射

| 发现 | 当前项目判断 | GF / 正确所有者 |
| --- | --- | --- |
| 不展示完整答案，以局部关系驱动发现 | 当前规则反馈丰富，缺分层提示与正式谜题关卡 | 提示语义/求解约束归项目 gameplay；UI 层复用既有 route/focus |
| 混入暂时无用的额外块 | 当前随机生成与稀疏棋盘可表达，但必须保持确定性 | 分发规则归关卡内容；随机选择必须走 `GFSeedUtility`，命令进入回放 |
| 七个房间与 Marathon 组织内容 | 当前六模式丰富，缺关卡目录与章节进度 | `GFLevelCatalog` 只在不制造平行内容真源时评估用于 pack/order/next；目标、解锁和完成归项目，进度写入 SaveGraph |
| 极简 HUD、棋盘信息自解释 | 当前部分页面信息权重接近 | 项目 UI 先做层级收敛；GF 只提供响应式、焦点和路由机制 |
| 放大相机跟随与一次按键响应 | 当前多平台输入基础强，需补谜题态验收 | 复用 `GFInputMappingUtility` 与 UI focus；相机语义归项目 |
| 多轨音乐同步修复 | 当前音频机制较完整但内容与跨状态验收浅 | 使用 `GFAudioUtility` / Bank；增加暂停、恢复、失焦和切场景测试 |
| 拼块重复/消失修复 | 当前确定性历史与保存优于一次性状态 | 稳定 ID、关卡实体归项目；SaveGraph 保持版本化恢复 |

## 可执行建议、风险与证据边界

- **P0**：为一个现有关卡/预设棋局实现三层提示，分别是相关区域、局部约束、单步建议；记录提示层级并验证撤销、回放和挑战资格。
- **P1**：建立 6—12 关的短章节目录，复用现有模式、拓扑与主题资产；完成奖励只解锁陈列/主题，不增加货币经济。
- **P2**：若采用分层音乐，增加轨道同步、暂停/恢复、设备失焦与场景重入验收，并验证弱设备降级。

Wilmot Works It Out 是商业产品，公开来源没有向本项目授权代码、美术、角色、音乐、音效或拼图复用；全部按不可复制处理。补丁日志证明修复结果，不证明内部原因或架构。视频只支持指定时间点的视觉事实，拾取手感、完整流程、音频和性能均未实测。
