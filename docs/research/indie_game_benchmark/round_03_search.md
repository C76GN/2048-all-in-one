## 本轮横向发现

1. **把资源 UI 变成空间规则。** GUNCHO 的弹巢方向、Card Thief 的光照/路径和 The Dungeon Beneath 的站位顺序都让辅助 UI 参与决策，而不是只在结算后报告数值。
2. **可接近性常来自约束而非功能堆叠。** Dreadrock 的单屏关卡、Forward 的单输入、Gun Rounds 的 one-button 与 Tinyfolks 的 1-bit 版式都先减少同时需要理解的动作。
3. **低保真不能替代性能证据。** 只有 Gun Rounds 开发日志明确提到性能与内存泄漏修复；其他候选只能作为克制表现参照，不能据像素风推断其帧预算。
4. **威胁提示属于核心规则呈现。** Militia 的 threatened-unit 更新、GUNCHO Puzzle Mode 的敌人顺序与 Dreadrock 的陷阱状态都表明预告应由同一规则计算产生。
5. **当前项目已有合适机制。** 确定性 seed、回放、真实响应式布局、GF 输入映射、Action Queue、Shader/音频 Utility 和 SaveGraph 足以承载多数方向；遗物、敌人、关卡、daily 资格和教程仍应由项目 Feature 拥有。

## 建议进入后续综合矩阵的增量

- P0 候选：为既有模式加入“下一动作影响预览”的只读原型；从同一规则结果生成，不建 UI 近似模拟器。
- P0 候选：把减少动态、单按钮/触控主动作和教程状态纳入同一可访问性验收矩阵。
- P1 候选：用现有 seed、回放和 SaveGraph 组合 daily/puzzle 资格，不另造随机数或时间线。
- P1 候选：把职业/装备式局内选择先做成独立实验模式；项目拥有内容语义，`GFTurnFlowSystem` 和 Action Queue 只承载机制。
- P1 候选：建立低端设备证据表，分开记录帧时间、输入首反馈、内存峰值、Shaderless/减少动态降级，避免用美术风格代替测量。
