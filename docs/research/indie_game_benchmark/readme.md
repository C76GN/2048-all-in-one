## 当前最强结论

- 当前项目的确定性、可变拓扑、历史/回放、主题和 GF 集成底座普遍强于样本；不应复制竞品架构或另建平行系统。
- 2026-07-23 已落地第一批工程切片：`TurnResult` 取代 `MoveData`，玩法 RNG 使用稳定 GF branch，回放 schema v2 记录规则指纹与逐回合 checkpoint 并在首个 OOS 阻断，固定 seed/规范 checksum 有回归；统一 feedback recipe、无障碍状态和 VFX 硬预算也已进入运行时。
- 当前工程优先项已收敛为快速输入 burst、六模式 × 多拓扑 seed catalog、gameplay/cosmetic 多配置等价测试，以及把性能目标矩阵升级为目标设备实测报告。
- 最优先的产品切片是交互式首次教程、高对比/色觉/屏幕阅读语义、daily challenge + 资格模型、行动后果预览，以及少量原创关卡目标。
- 追加轮次 A 进一步确认：有限且公开的下一资源、承诺前的合法性/得分预演、对象附近的配方说明，比继续横向增加合并规则更能形成可解释的纵向深度。
- 追加轮次 B 进一步确认：相机/HUD 必须服从 canonical 玩法参照系；多格路径先草拟再原子提交；短局深度优先来自单屏手工课程、少量 build 选择和公开意图，而不是新增平行框架系统。
- 网格肉鸽深度应从“单敌人 + 单障碍”或“3 个局内升级”的可回放垂直切片开始；遗物、卡牌、敌人和经济都属于项目业务，不属于 GF。
- GF 运行时机制已覆盖 turn/action queue、seed/clock、history/SaveGraph、Shader/预热、音频、输入、对象池和 diagnostics。当前框架反馈主要是 capability 目录关键词/`primary_classes` 的发现性，不是重造 API。

## 安全与采用边界

- 第三方仓库只位于 `E:\_workspace\Godot Project\_research\2048-benchmark`，均以 detached HEAD 固定完整 SHA；未构建、运行、安装依赖或初始化 submodule。
- 本项目只提交研究 Markdown；没有提交第三方仓库、缓存、截图、视频、代码、Shader、音频或美术素材。
- MIT 等宽松代码许可证不自动覆盖素材、字体、商标与依赖。GPL/AGPL/CC-BY-SA 或未公开源码样本只用于观察机制、分类语言与测试方法。
- 本批研究不修改游戏代码或 `addons/gf`。任何落地项都应另开 issue/分支/PR，并以行动建议中的验收条件为边界。
