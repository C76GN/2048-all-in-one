## 方法与安全边界

- 先与现有 87 个候选及 20 份深度报告按名称、系列和产品链接去重，再收录 12 个新对象。
- 候选至少连接作者/工作室官网、作者 itch.io、开发者或发行商新闻、官方 press kit、官方商店页之一；本轮没有把搜索摘要或社区页面当作关键结论。
- 本轮只访问网页和开发者/发行商视频，没有下载、安装、构建或运行任何游戏、仓库、素材或脚本。
- 下表证据等级沿用 [研究方法](methodology.md)：B 为作者、开发者或发行商可控制的产品页/开发日志/官方商店页。视频逐帧观察另见 [R2-A 视频台账](videos_round_02.md)，属于 A 级画面证据。
- 所有候选均未见向本项目授予代码、Shader、音频或美术复用许可；默认仅观察产品设计，按商业素材不可复用处理。

## 深度分析选择

本轮选择四个互补对象，而不是四个相似的六角拼图：

1. [Dorfromantik](projects/game_r2_dorfromantik.md)：有限牌堆、边缘匹配和月度挑战。
2. [Stacklands](projects/game_r2_stacklands.md)：卡牌堆叠即配方、生产、战斗与空间管理。
3. [Wilmot Works It Out](projects/game_r2_wilmot_works_it_out.md)：无盒图拼块、混合到货与极简界面。
4. [Freshly Frosted](projects/game_r2_freshly_frosted.md)：有限空间路径构造、连续确定性模拟和分层教学。

## 对当前项目的首轮比较

| 横向发现 | 当前项目基线 | GF 与正确所有者 |
| --- | --- | --- |
| 摆放前先公开邻接得分、合法性或路线结果 | 当前移动结果清楚，但没有“承诺前的空间方案预览”；棋盘编辑器是合适试验面 | 预览数据和规则属于 gameplay/board_editor；UI/焦点复用现有 GF，不新增框架规则系统 |
| 有限牌堆、关卡箱或岛屿给单局明确终点 | 当前六模式以分数/目标值为主，缺正式谜题关卡与短目标 | 关卡目录可用项目内容注册/包机制，进度进入 SaveGraph；关卡语义仍属项目 |
| 极少主动作也能产生高密度结果 | 当前反馈技术丰富，但部分页面信息权重接近 | 用项目 motion/feedback Profile 和既有 Action Queue 收敛层级，不增加平行 Tween 管理器 |
| 重排、拖放、旋转需要暂停、撤销和多输入同等可靠 | 当前确定性历史、抽象输入和真实响应式布局强，可直接复用 | `GFInputMappingUtility`、命令历史、UI route/focus 提供机制；拖放语义和可访问性策略属项目 |
| 月度/每日共享挑战提高有限内容复用率 | 当前已有手动 seed、稳定 hash 和回放，但没有日/月挑战资格模型 | `GFSeedUtility`、SaveGraph、回放和平台 Adapter 足够；日历、规则版本、奖励和线上权威属项目 |
| 卡牌/拼块数量增长会把命中、遮挡与状态身份变成风险 | 当前稀疏棋盘和池化有基础，但没有任意重叠对象压力场景 | 先在项目中做有界索引、对象身份和性能基准；没有跨项目证据前不提 GF 运行时缺口 |

## 轮次结论与证据限制

本轮最强的新方向是“摆放前可解释、摆放后有仪式、整局有有限资源或明确终点”。这比继续增加数值合并规则更能补当前项目的纵向深度。优先级最高的后续验证应是一个只读摆放预览和一个可回放的有限关卡切片；两者都必须复用现有确定性状态、撤销/重做与输入抽象。

候选产品页可以证明公开功能、平台和作者表达，不能证明内部算法、具体 Shader、帧率、内存或音频混音。本轮深度报告中的画面事实来自官方视频时间点；所有性能建议均明确区分“官方披露”“画面观察”和“对本项目的风险推断”。
